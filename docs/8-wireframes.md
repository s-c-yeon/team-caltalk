## 1. 읽는 방법 (표기 범례)

와이어프레임은 ASCII 박스 다이어그램으로 표현한다. 아래 표기를 공통으로 사용한다.

| 표기 | 의미 |
|---|---|
| `┌─┐ │ └─┘` | 화면/패널/카드 등 영역의 경계 |
| `[ 버튼 ]` | 클릭 가능한 버튼 |
| `[______]` | 텍스트 입력 필드 |
| `( )` `(●)` | 라디오 버튼(미선택/선택) |
| `▾` | 드롭다운/선택 컨트롤 |
| `*` | 현재 선택되었거나 활성화된 상태(예: 선택된 날짜, 선택된 탭) |
| `< >` | 이전/다음 이동 컨트롤 |
| `…` | 반복되는 항목의 생략 |

각 화면은 **화면 ID / 제목 / 관련 SC·EX·FR ID / ASCII 와이어프레임 / 보충 설명(1~3문장)** 순서로 기술한다. 보충 설명은 다이어그램이 담지 못한 상호작용만 다루며, 다이어그램 내용을 글로 반복하지 않는다.

### WF-04. 일정 생성/수정 폼

- **관련 ID**: SC-02, SC-03 / FR-04, FR-05, 도메인정의서 §8

```
┌───────────────────────────────────────────────────────┐
│  일정 생성                                    [ X 닫기]│
├───────────────────────────────────────────────────────┤
│  제목                                                  │
│  [_____________________________________________]      │
│   ※ 1~100자, 공백만으로는 입력 불가 (§8)               │
│                                                         │
│  시작 일시                     종료 일시                │
│  [2026-07-22 14:00]     [2026-07-22 15:00]             │
│   ※ 시작 < 종료 (같거나 역전 불가, §8)                  │
│                                                         │
│  대상 (단일 선택)                                       │
│   (●) 팀 전체                                          │
│   ( ) 특정 팀원 1명   [ 팀원 선택 ▾ ]                   │
│   ※ 팀 전체 또는 특정 팀원 1명 중 하나만 선택 가능 (§8) │
│                                                         │
│              [ 취소 ]   [ 삭제 ]*   [   저장   ]        │
│                          * 수정 화면에서만 노출          │
└───────────────────────────────────────────────────────┘
```

`[+ 일정 추가]`로 진입하면 "일정 생성" 제목·빈 필드로, 기존 일정 클릭으로 진입하면 "일정 수정" 제목·기존 값이 채워진 상태 및 `[삭제]` 버튼이 함께 표시된다(SC-03). 삭제 클릭 시 별도의 확인 다이얼로그가 뜬 뒤 캘린더에서 즉시 제거된다. 이 폼은 팀장 전용이며(BR-03), SC-07에서 팀장이 변경 요청을 반영할 때도 동일한 화면을 그대로 재사용한다.

### WF-07. 다중 팀 전환 UI

- **관련 ID**: SC-09, EX-03 / FR-13

```

│                                 │            │  └───────────────────────┘    │
└───────────────────────────────┘            └───────────────────────────────┘

전환 후 (팀 B, 팀장 역할) — 화면 전체가 교체됨. 팀 A 일정과 병합되지 않음.
┌───────────────────────────────────────────────────────┐
│ [팀 선택: 팀 B ▾]                        [팀장] ...    │
├───────────────────────────────────────────────────────┤
│  (WF-02 팀장 뷰와 동일)                                 │
│  · [+ 일정 추가] 등 CUD 진입점 노출                      │
│  · 팀 A의 일정/채팅은 이 화면에 전혀 표시되지 않음        │
└───────────────────────────────────────────────────────┘
```

`TeamSwitcher`에서 팀을 바꾸면 캘린더·채팅 패널을 포함한 화면 전체(WF-02/WF-03)가 선택한 팀의 데이터로 교체되며, 현재 사용자의 역할(팀장/팀원)에 따라 CUD 진입점 노출 여부도 함께 바뀐다(SC-09). 두 팀의 일정을 한 캘린더에 겹쳐 보여주는 통합 뷰는 의도적으로 제공하지 않는다(BR-11, EX-03).

### WF-08. 팀 설정 / 탈퇴 화면

- **관련 ID**: EX-01, EX-04 / 도메인정의서 §3.3, §6.1, §6.4

```
정상 상태 (다른 팀장 존재, 또는 팀원으로서 탈퇴 시도)
┌─ 팀 설정 (TeamSettings) ───────────────────────────────┐
│  팀 이름: 개발1팀                                       │
│                                                          │
│  팀원 목록                                               │
│   · 홍길동 (팀장)                                        │
│   · 김철수 (팀원)                                        │
│   · 이영희 (팀원)                                        │
│                                                          │
│  ─────────────────────────────────────────────────────  │
│                                    [   이 팀에서 탈퇴   ] │
└──────────────────────────────────────────────────────┘

차단 상태 — 팀장이 유일한 팀장인 팀에서 탈퇴를 시도 (EX-01)
┌─ 팀 설정 (TeamSettings) ───────────────────────────────┐
│  팀 이름: 개발1팀                                       │
│                                                          │
│  팀원 목록                                               │
│   · 홍길동 (팀장) — 유일한 팀장                          │
│                                                          │
│  ┌────────────────────────────────────────────────┐   │
│  │ ⚠ 탈퇴할 수 없습니다.                             │   │
│  │  이 팀에는 다른 팀장이 없습니다.                   │   │
│  │  먼저 다른 팀원을 팀장으로 지정한 뒤 다시           │   │
│  │  시도해주세요.                                    │   │
│  └────────────────────────────────────────────────┘   │
│                                    [ 이 팀에서 탈퇴 ]    │
│                                     (비활성화됨)         │
└──────────────────────────────────────────────────────┘
```

탈퇴 버튼 클릭 시 시스템은 항상 확인 절차를 거친 뒤 처리하되, 팀장이 유일한 팀장인 상태에서 탈퇴를 시도하면 위와 같은 차단 메시지를 노출하고 탈퇴를 진행하지 않는다(BR-09, EX-01). 팀원이 탈퇴하거나, 팀장이 마지막으로 남아있던 소속 팀에서(다른 팀장 지정 후) 탈퇴하는 경우에는 정상 처리되며, 마지막 소속 팀에서 탈퇴한 사용자는 계정은 유지된 채 "소속 팀 없음" 상태로 재로그인하게 된다(EX-04) — 이 상태 자체는 로그인 직후 캘린더 화면이 "소속된 팀이 없습니다" 안내로 대체되는 것으로 처리되며 별도 화면으로 그리지 않는다.

### WF-09. 반응형(모바일) 캘린더 메인 화면

- **관련 ID**: SC-08 / FR-17 (WF-02의 모바일 재배치 예시)

```
데스크톱(WF-02): 캘린더·채팅 패널 좌우 병렬 배치
┌───────────────────────────────────┐
│  캘린더 영역   │   채팅 패널        │
│  (넓음)        │   (고정 폭)        │
└───────────────────────────────────┘

모바일: 좌우 병렬 대신 상단 탭으로 전환하여 한 번에 하나씩 표시
┌─────────────────────────┐   ┌─────────────────────────┐
│ Team CalTalk   ☰ 메뉴    │   │ Team CalTalk   ☰ 메뉴    │
├─────────────────────────┤   ├─────────────────────────┤
│ [ 캘린더* ] [  채팅  ]   │   │ [ 캘린더  ] [  채팅* ]   │
├─────────────────────────┤   ├─────────────────────────┤
│ [월*][주][일] ◀ 7월 ▶   │   │  채팅 · 2026-07-22       │
│                          │   │ ─────────────────────── │
│  일 월 화 수 목 금 토    │   │ 09:20 팀원A: 3시로...    │
│         1  2  3  4      │   │        [변경 요청]       │
│  5  6  7  8  9 10 11    │   │ 09:30 팀장: 확인했습니다 │
│    [정기회의]            │   │                          │
│ 19 20 [21][22*][23] 24  │   │ ─────────────────────── │
│    [팀회의 14:00]        │   │ (●)일반 ( )변경요청      │
│                          │   │ [________________][전송]│
│  [ + 일정 추가 ]*        │   │                          │
└─────────────────────────┘   └─────────────────────────┘
  탭: "캘린더" 선택 상태          탭: "채팅" 선택 상태
  * 팀장 뷰에서만 노출            (날짜 선택은 캘린더 탭에서 유지됨)
```

데스크톱에서는 캘린더와 채팅이 한 화면에 나란히 배치되지만(WF-02/WF-03), 모바일 화면 폭에서는 두 영역을 동시에 보여주기 어려우므로 상단 `[캘린더]`/`[채팅]` 탭으로 전환하는 방식을 사용한다. 캘린더 탭에서 날짜를 선택한 뒤 채팅 탭으로 이동하면 그 선택된 날짜의 채팅이 열려, 데스크톱과 동일한 캘린더-채팅 연동 동작(BR-07)이 유지된다. 팀장 뷰/팀원 뷰의 CUD 버튼 노출 차이(WF-02 vs WF-03)도 모바일에서 동일하게 적용된다.
